\section{Self-Predictive Representations}
\label{sec:spr-architecture}

In the low-data regime, the standard TD loss provides a sparse and
noisy learning signal -- the agent may see only a handful of
rewarding transitions across 100K environment steps. Self-Predictive
Representations (SPR)~\citep{schwarzer2021dataefficient} address
this by adding an auxiliary objective that encourages the encoder to
learn representations from which future latent states can be
predicted. This gives the encoder a dense, self-generated training
signal on every transition, independent of reward.

\smallskip
\Cref{fig:spr-architecture} illustrates the architecture. The
online encoder $f_\theta$ -- the same Nature CNN used for
Q-learning (\cref{subsec:network-architecture}) -- produces spatial
feature maps $z_t$ from the current observation. A convolutional
transition model $h$ takes $z_t$ and the agent's action $a_t$ to
predict the next latent state $\hat{z}_{t+1}$. This process repeats
for $K$ steps, producing predictions $\hat{z}_{t+1}, \ldots,
\hat{z}_{t+K}$ from a single starting state. Each predicted state
is passed through a projection head $g$ and a prediction head $q$
to produce the final prediction vector.

\smallskip
The target for each prediction comes from a separate path: the
actual future observation $s_{t+k}$ is fed through an exponential
moving average (EMA) copy of the encoder $\bar{f}_\theta$ and an
EMA projection head $\bar{g}$. The prediction head $q$ exists only
on the online side, creating the asymmetry that prevents
representation collapse -- without it, both paths could converge to
a trivial constant output~\citep{grill2020bootstrap}.

\smallskip
The SPR loss is the negative cosine similarity between the online
prediction and the target representation, averaged over valid
predictions (those that do not cross episode boundaries):
%
\begin{equation}
\label{eq:spr-loss}
\mathcal{L}_{\text{SPR}}
  = -\frac{1}{N}\sum_{k=1}^{K}\sum_{b=1}^{B}
    m_{k,b}\;\frac{\tilde{y}_{k,b} \cdot \bar{y}_{k,b}}
    {\lVert\tilde{y}_{k,b}\rVert\;\lVert\bar{y}_{k,b}\rVert}
\end{equation}
%
where $\tilde{y}$ is the online prediction, $\bar{y}$ is the target,
$m_{k,b}$ masks out predictions that cross episode boundaries, and
$N$ is the number of valid predictions. The total training loss
combines the standard TD objective with the SPR loss:
%
\begin{equation}
\label{eq:total-loss}
\mathcal{L} = \mathcal{L}_{\text{TD}}
  + \lambda\,\mathcal{L}_{\text{SPR}}
\end{equation}
%
with $\lambda = 2.0$ following the original
authors~\citep{schwarzer2021dataefficient}. Because $f_\theta$ is
shared between Q-learning and the SPR objective, the auxiliary loss
directly shapes the representations used for action selection.

\indent A key design choice is that the transition model operates on
the spatial feature maps ($64 \times 7 \times 7$ after Conv3) rather
than the flattened FC output. This preserves spatial structure,
allowing the model to condition on actions via spatial broadcasting
-- the one-hot action vector is replicated across all spatial
positions and concatenated along the channel dimension. The
transition model itself is a two-layer convolutional network with
batch normalization, keeping the same spatial resolution as its
input.

\smallskip
The EMA target encoder is updated after each training step:
$\bar{\theta} \leftarrow \tau\,\bar{\theta} + (1 - \tau)\,\theta$,
where $\tau$ controls the update speed. In our implementation,
$\tau = 0.99$ when augmentation is disabled and $\tau = 0.0$ (direct
copy) when augmentation is enabled, following the finding that data
augmentation provides sufficient regularization on its own. We set
the prediction depth to $K = 5$ steps for all experiments.
\Cref{tab:spr-hyperparams} summarizes all SPR hyperparameters.

\section{Interpretability Methods}
\label{sec:interpretability-methods}

This section describes the methods we use to analyze what the
encoder has learned and how its representations vary across
experimental conditions.

\subsection{Linear Probing}
\label{subsec:linear-probing}

To evaluate what information the encoder captures, we use linear
probing~(\cref{fig:linear-probe}). The encoder $f_\theta$ is first
trained end-to-end via reinforcement learning, then its weights are
frozen. A single linear layer is trained on the frozen
representations to predict known game-state properties. Because the
probe is restricted to a linear mapping, any structure it recovers
must already be present in the representation -- the probe cannot
learn new features of its own. Probe accuracy therefore measures
what information $f_\theta$ has made linearly accessible.

\clearpage
\input{figures/linear-probe}
\clearpage

\indent This approach lets us compare representations across experimental
conditions (e.g., with and without SPR, or across agent
architectures) on a common basis: we hold the probing procedure
fixed and vary only the encoder that produced the representations.
Differences in probe accuracy reflect differences in what the
encoder has learned to represent, not differences in probe capacity.

\subsection{Probe Design}
\label{subsec:probe-design}

Each probe is a single fully connected layer mapping from the
representation $h_t \in \mathbb{R}^d$ to a target
$y \in \mathbb{R}^k$, trained with mean squared error for
continuous targets and cross-entropy for categorical targets. We
train probes on states sampled from completed evaluation episodes,
using the same set of states across all conditions to ensure
comparability.

\smallskip
The probe targets are game-state properties that a capable agent
should represent: object positions, score-relevant features, and
spatial relationships. The specific targets vary by game and are
defined in \cref{sec:experiment-design}. For each target, we report
probe accuracy (classification) or $R^2$ (regression) on a held-out
test set. \Cref{tab:probe-training} summarizes the probe training
configuration.

\subsection{Latent Space Visualization}
\label{subsec:latent-viz}

While linear probes quantify what information a representation
contains, they do not reveal how that information is organized.
Two encoders may achieve similar probe accuracy yet structure their
representations very differently -- one separating game states into
tight clusters, the other spreading them across a diffuse manifold.
Dimensionality reduction techniques let us visualize this structure
directly.

We apply both t-SNE~\citep{vandermaaten2008visualizing} and
UMAP~\citep{mcinnes2018umap} to the encoder's representations,
projecting them from $\mathbb{R}^d$ to $\mathbb{R}^2$ for
visualization. Using both methods guards against artifacts: t-SNE
emphasizes local neighborhood structure and tends to reveal
well-separated clusters, while UMAP better preserves global
relationships and distances between clusters. Patterns that appear
consistently under both methods are more likely to reflect genuine
structure in the representation.

\smallskip
For each experimental condition and each probe attachment point
(\cref{tab:probe-attachments}), we collect representations from the
same evaluation episodes used for probing. Each point in the
resulting scatter plot corresponds to a single state $s_t$, colored
by a game-state property (e.g., object position, score) to reveal
whether the encoder organizes its representations around
task-relevant features. We generate side-by-side plots across the
four experimental conditions to compare how augmentation and SPR
reshape the latent space.

Comparing visualizations across layers shows whether structure
emerges gradually through the network or appears abruptly after a
specific layer. Comparing across conditions reveals whether SPR or
augmentation produces more structured representations, complementing
the quantitative probe results with a qualitative view of
representation geometry.

\smallskip
Several caveats guide our interpretation of these plots. In t-SNE,
the relative sizes of clusters are not meaningful -- a tight cluster
and a diffuse one may contain the same density of points in the
original space~\citep{vandermaaten2008visualizing}. Distances
between clusters are also unreliable; t-SNE can place unrelated
groups at arbitrary separations depending on the perplexity setting.
UMAP preserves global structure more faithfully, but distances
within dense regions can still be compressed. For both methods,
results are sensitive to hyperparameters (perplexity for t-SNE,
number of neighbors for UMAP), so we report these values explicitly
(\cref{tab:viz-params}) and verify that our qualitative conclusions
hold across a range of settings. We treat these visualizations as
complementary to the probe results, not as standalone evidence.

\smallskip
\input{figures/latent-viz-guide}

\Cref{fig:viz-guide} illustrates the two extremes. In a structured
representation~(\cref{fig:viz-guide-good}), points with the same
game-state property form distinct clusters, indicating the encoder
organizes states around task-relevant features. In an unstructured
representation~(\cref{fig:viz-guide-bad}), the same coloring reveals
no grouping -- game-state properties are not separable in the
projected space. In practice, results fall between these extremes,
and the degree of clustering provides a qualitative measure of how
well the encoder captures the target property.

\subsection{Transition Model Analysis}
\label{subsec:transition-analysis}

SPR trains a transition model $g_\phi$ that predicts future latent
representations given the current representation and intervening
actions~(\cref{sec:spr-architecture}). During training, this model
drives the self-predictive loss that shapes the encoder's
representations. We analyze the transition model directly to
understand what kind of dynamics it has learned and how prediction
quality relates to representation quality.

\smallskip
For each SPR-enabled condition, we collect trajectories from
evaluation episodes and compute the cosine similarity between the
transition model's predicted representation
$\hat{h}_{t+k} = g_\phi(h_t, a_t, \ldots, a_{t+k-1})$ and the
encoder's actual representation $h_{t+k} = f_\theta(s_{t+k})$ at
each prediction horizon $k = 1, \ldots, K$, where $K$ is the SPR
prediction depth used during training. This produces a prediction
quality curve showing how faithfully the model anticipates future
representations as the horizon increases.

\smallskip
We examine this curve along two dimensions. First, the rate of
degradation over $k$ steps reveals how far into the future the
transition model remains reliable -- a slow decay suggests the model
has learned robust dynamics, while rapid decay indicates it relies
on short-range correlations. Second, we condition the analysis on
game-state properties (e.g., object proximity, score changes) to
test whether prediction quality varies with the complexity of the
underlying transition. States involving collisions or reward events
may be harder to predict than smooth trajectories.

\smallskip
This analysis applies only to the two experimental conditions where
SPR is enabled. By comparing transition model quality between the
SPR-only and augmentation+SPR conditions, we can assess whether
data augmentation improves or degrades the dynamics model. Combined
with the probe results from \cref{subsec:linear-probing}, this
reveals whether better predictions correspond to more informative
representations.
