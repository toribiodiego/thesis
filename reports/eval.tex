\section{Part 1: DQN Isolation Study (Thesis 4.1)}
% ================================================================

\noindent These figures answer: \textit{Does SPR improve vanilla
DQN, and how does it interact with augmentation?}
Reproduction: Appendix~\ref{app:perf-figs}.

\subsection*{A1. Learning Curves}

\noindent\textit{Purpose.} Does SPR or augmentation change how
vanilla DQN learns over 100K steps? One subplot per game shows
the evaluation return at each checkpoint for all four DQN
conditions.

\begin{itemize}[leftmargin=2em, itemsep=0.1em, topsep=0.3em]
  \item Four conditions: DQN, +Aug, +SPR, +Both
  \item Shaded bands: $\pm 1$ std (single seed) or cross-seed
    confidence interval (multi-seed)
  \item Status: example generated from Boxing data (seed 42);
    full 6-game grid after batch runs
\end{itemize}

\begin{figure}[H]
\centering
\includegraphics[width=0.82\textwidth]{../output/plots/boxing_learning_curves.png}
\caption{Example: Boxing, seed 42. Full version uses 6-game grid.}
\end{figure}


\subsection*{A2. Intervention Lift}

\noindent\textit{Purpose.} How much does adding SPR or
augmentation change the final score? Shows the magnitude of each
intervention's effect as a score delta -- positive means it
helped, negative means it hurt.

\begin{itemize}[leftmargin=2em, itemsep=0.1em, topsep=0.3em]
  \item Left panel: score change from adding SPR (with and
    without augmentation)
  \item Right panel: score change from adding augmentation
    (with and without SPR)
  \item Status: example from Boxing (seed 42); full version
    groups all 6 games
\end{itemize}

\begin{figure}[H]
\centering
\includegraphics[width=0.82\textwidth]{../output/plots/boxing_spr_aug_lift.png}
\caption{Example: Boxing, seed 42. Full version groups all 6 games.}
\end{figure}


\subsection*{A3. Interaction Plot}

\noindent\textit{Purpose.} Do augmentation and SPR help
independently, or does one change the other's effect? Plots
absolute scores for all four conditions to reveal whether the
combination is additive, redundant, or interfering.

\begin{itemize}[leftmargin=2em, itemsep=0.1em, topsep=0.3em]
  \item Parallel lines = independent effects
  \item Crossing lines = the two interventions interact
  \item Status: example from Boxing (seed 42); full version has
    one subplot per game
\end{itemize}

\section{Part 2: Rainbow Capability Comparison (Thesis 4.2)}
% ================================================================

\noindent These figures answer: \textit{Does SPR need a capable
base agent to produce useful representations?}
Reproduction: Appendix~\ref{app:perf-figs}.

\subsection*{B1. Learning Curves -- Rainbow Comparison}

\noindent\textit{Purpose.} Does SPR need a stronger base agent
to be effective? Compares learning trajectories for DQN+SPR,
Rainbow, and Rainbow+SPR on the same six games -- if SPR only
helps Rainbow, the base agent's learning infrastructure matters
more than the representation objective itself.

\begin{itemize}[leftmargin=2em, itemsep=0.1em, topsep=0.3em]
  \item Three conditions: DQN+SPR, Rainbow, Rainbow+SPR
  \item 6 subplots (one per game), same format as A1
  \item Shaded bands: $\pm 1$ std or cross-seed confidence interval
  \item Status: pending -- requires completed Rainbow and
    Rainbow+SPR batch runs
\end{itemize}

\vspace{0.5em}
\subsection*{B2. SPR Lift Comparison}

\noindent\textit{Purpose.} How much does SPR improve DQN compared
to how much it improves Rainbow? Pairs the SPR lift on each base
agent side by side per game -- if the bars are consistently larger
for Rainbow, SPR's value depends on what the base agent provides.

\begin{itemize}[leftmargin=2em, itemsep=0.1em, topsep=0.3em]
  \item Paired bars per game: (DQN+SPR $-$ DQN) vs
    (Rainbow+SPR $-$ Rainbow)
  \item Same color scheme as A2 (light fill, dark edge)
  \item Status: pending -- requires completed Rainbow+SPR runs
\end{itemize}

are close -- context for interpreting where SPR's contribution
matters.

\begin{itemize}[leftmargin=2em, itemsep=0.1em, topsep=0.3em]
  \item x = DQN final score, y = Rainbow final score
  \item Diagonal line marks parity (above = Rainbow better)
  \item Status: pending -- requires completed Rainbow runs

\begin{itemize}[leftmargin=2em, itemsep=0.1em, topsep=0.3em]
  \item 6 games (rows) $\times$ 6 conditions (columns)
  \item Multi-seed mean $\pm$ confidence interval
  \item Status: pending -- requires all batch runs complete

\noindent Remaining pending figures:

\subsection*{B5. Human-Normalized Scores}

\noindent\textit{Purpose.} How do our conditions compare when
scores are standardized across games? Raw scores are not
comparable between games (Boxing ranges $\pm$100, Crazy Climber
reaches 30,000+), so normalizing by human and random baselines
puts all games on the same scale.

\begin{itemize}[leftmargin=2em, itemsep=0.1em, topsep=0.3em]
  \item Same structure as B4, scores normalized:
    HNS = (agent $-$ random) / (human $-$ random)
  \item Requires published human and random baselines per game
  \item Status: pending -- requires batch runs and baseline lookup
\end{itemize}

\subsection*{B6. Aggregate Performance}

\noindent\textit{Purpose.} Which conditions improve performance

SPR and augmentation help across the board or only on specific
games.

\begin{itemize}[leftmargin=2em, itemsep=0.1em, topsep=0.3em]
  \item One bar per condition, height = mean HNS across 6 games
  \item Error bars: cross-game standard deviation
  \item Status: pending -- derived from B5

\section{Part 3: Interpretability Analysis (Thesis 4.3)}
% ================================================================

\noindent These figures answer: \textit{What do the learned
representations actually encode, and does that change between
DQN and Rainbow?}

\noindent All figures in this section require the feature
extraction and probing pipeline. Checkpoints with
\texttt{spr\_state} will be available after batch runs.
Reproduction: Appendix~\ref{app:interp-figs}.

\subsection*{C1. Probe Accuracy}

\noindent\textit{Purpose.} Do SPR-trained encoders learn more
about the game state than vanilla encoders? Linear probes on
frozen encoder outputs test whether features like player position
and score are decodable -- higher accuracy means the encoder
captures more game-relevant structure.

\begin{itemize}[leftmargin=2em, itemsep=0.1em, topsep=0.3em]
  \item Grouped bars: F1 per game, per condition, per probe target
  \item Probe targets: player position, score, lives (from ALE RAM)
  \item All 6 conditions compared on same axes
  \item Status: pending -- requires feature extraction and
    probing pipeline
\end{itemize}

\vspace{0.5em}
\subsection*{C2. Probe Accuracy Over Training}

\noindent\textit{Purpose.} When during training do the
representations become useful? Probing at multiple checkpoints
shows whether feature quality improves steadily, plateaus early,
or only emerges late -- and whether DQN and Rainbow develop
useful representations on different timelines.

\begin{itemize}[leftmargin=2em, itemsep=0.1em, topsep=0.3em]
  \item x = checkpoint step, y = probe F1, lines per condition
  \item One plot per game, all 6 conditions overlaid
  \item 10 checkpoints (every 40K frames) for new runs,
    4 checkpoints for existing DQN/+Aug runs
  \item Status: pending -- requires feature extraction and
    probing at each checkpoint
\end{itemize}

\vspace{0.5em}
\noindent Remaining pending figures:

\begin{description}[leftmargin=2em, itemsep=0.4em]
  \item[C3. Latent PCA / t-SNE] 2D scatter colored by probe
    target. Grid: rows = conditions, columns = games. Shows
    structural differences in representations.
  \item[C4. Transition model similarity] x = prediction step K

  \item[C5. Metric correlation] Heatmap: Rainbow metrics vs games,

\end{description}

\section{Regeneration}
\label{app:regen}

All figures can be reproduced from run data on Google Drive.

\subsection{Downloading run data}
\label{app:download}

\begin{verbatim}
bash scripts/pull-run.sh --all              # all runs
bash scripts/pull-run.sh --group base       # vanilla DQN
bash scripts/pull-run.sh --group spr        # DQN+SPR
bash scripts/pull-run.sh --group rainbow    # Rainbow
bash scripts/pull-run.sh --all --no-checkpoints  # skip .pt files
bash scripts/pull-run.sh --list             # list available
\end{verbatim}

Run directories are documented in
\texttt{experiments/dqn\_atari/runs/INDEX.md}.

\subsection{Performance figures (A, B, D1--D7)}
\label{app:perf-figs}

\begin{verbatim}
python scripts/plot_learning_curves.py  # learning curves
python scripts/plot_results.py          # score tables, bar charts
\end{verbatim}

\noindent Both scripts read \texttt{evaluations.csv} and
\texttt{training\_steps.csv} from local run directories.

\subsection{Post-hoc assessment}
\label{app:reeval}

\begin{verbatim}
python scripts/reeval_checkpoints.py --device cuda  # fast
python scripts/reeval_checkpoints.py --record-video # with video
python scripts/reeval_checkpoints.py --force        # re-run all
\end{verbatim}

\noindent Run on Colab T4, not locally. See
\texttt{reports/eval-pipeline.tex} for details.

\subsection{Interpretability figures (C1--C5)}
\label{app:interp-figs}

Requires interpretability infrastructure. Scripts will be:

\begin{verbatim}
python scripts/extract_features.py      # feature extraction
python scripts/run_probes.py            # linear probes
python scripts/plot_latent_space.py     # latent visualization
python scripts/analyze_transitions.py   # transition model
python scripts/correlate_metrics.py     # metric correlation
\end{verbatim}
