\chapter{Methods}
\label{ch:methods}

Our work examines how reinforcement learning agents develop their
internal features, what determines how well those features capture
the task, and what can be done to improve them. \Cref{sec:agents}
introduces two agents that develop features differently, and
\cref{sec:experimental-design} defines the experiment that tests
what accounts for the difference. \Cref{sec:analysis-methods}
describes how we analyze what the features capture.

\section{Agents}
\label{sec:agents}

Data-Efficient Rainbow (DER) \citep{vanHasselt2019} established the
first baseline for learning from limited
data.\footnote{SimPLe \citep{kaiser2020model} introduced the
Atari-100K benchmark with a model-based approach.
\citet{vanHasselt2019} showed that tuning Rainbow's hyperparameters
alone matched its performance without architectural changes.}
The agent observes the game, takes actions, and uses the game score
as its reward, learning which choices work and developing its
representations from that experience.

\smallskip
Self-Predictive Representations (SPR) \citep{schwarzer2021dataefficient}
consistently outperforms DER across the Atari-100K
benchmark.\footnote{On the 26-game Atari-100K benchmark, SPR achieves
a median human-normalized score of 0.510 compared to DER's
0.161~\citep{schwarzer2021dataefficient}.}
SPR trains the agent to predict its own future representations,
adding an objective beyond the game score. A predictor, a transition
model, and a prediction loss enable this
prediction.\footnote{We use the JAX reimplementation from
\citet{nikishin2022primacy}, which uses 512 dimensions and a separate
projection layer. The original PyTorch version uses 256 dimensions.}
We call these the \emph{latent prediction system} and apply it to
both agents to understand what shapes more capable agents.
